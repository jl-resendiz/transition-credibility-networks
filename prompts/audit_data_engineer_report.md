### GAP 2 (LOW): `robustness_conley_se.R` not in `run_all.py`

The R script produces `results/metrics/strategy2_conley_se.md`. The manuscript cites Conley SE results (Section 4.6: "t = -4.16 at 1000km"). There is no Python equivalent (`strategy2_conley_se.py` does not exist). The R script requires `fixest` and `data.table` packages.

Since `run_all.py` only runs Python scripts, the R script cannot be directly integrated. Two options:
1. Add a subprocess call: `subprocess.run(['Rscript', 'robustness_conley_se.R'], cwd=SRC)`
2. Document it as a manual step in README.

**Recommendation:** Add as an optional step with a `--with-r` flag, or document as manual.

### GAP 3 (LOW): `update_event_dates.py` and `update_announcement_dates.py` not in `run_all.py`

These scripts patch `coal_retirement_events.csv` with manually researched exact dates and announcement dates. They run after `build_retirement_events.py` and before analysis scripts. Without them, the event file may lack precise dates.

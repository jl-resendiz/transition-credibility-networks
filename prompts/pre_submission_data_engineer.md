# Pre-Submission Audit: Data Engineer

You are a data engineer reviewing the pipeline infrastructure before journal submission. The paper uses a stdlib-only Python pipeline (no pandas, numpy, scipy) with openpyxl as the only non-stdlib dependency.

## Your Task

Ensure the pipeline is complete, reproducible, well-documented, and ready for a replication package.

## Specific Checks

### 1. Pipeline Orchestrator (`src/run_all.py`)

Verify that `run_all.py` includes ALL scripts that produce outputs cited in the manuscript:
- Compare the list of scripts in `run_all.py` against all `strategy*.py` and `build_*.py` files in `src/`
- KNOWN GAP: `strategy2_bartik_shiftshare.py` exists but is NOT in run_all.py. It produces `results/metrics/strategy2_bartik_shiftshare.md` which is cited in the manuscript (Bartik t=-5.16, Oster δ*=20.8). Add it.
- KNOWN GAP: `strategy2_conley_se.py` — check if it exists and whether it should be in run_all.py (Conley SEs are cited: t=-4.16 at 1000km).
- Check: `summary_statistics.py` — is it in run_all.py?
- Check for any script in `src/` that is NOT in run_all.py but produces files in `results/`

### 2. Dependency Order

Verify that build scripts run in correct dependency order:
```
parse_gem → match_gem_compustat → build_fundamentals, build_fuel_matrix, build_ets_matrix, build_weight_matrix → build_time_varying_alpha → compute_returns → build_retirement_events, build_coal_phaseout_events, build_eia860_announcement_events → summary_statistics
```
Check that no analysis script depends on another analysis script's output (they should all be independent given the derived data).

### 3. Input/Output Manifest

For each script in `src/`, document:
- Input files (what it reads)
- Output files (what it writes)
- Verify that every output file cited in the manuscript is produced by exactly one script
- Verify that no script writes to a path that another script also writes to (no collisions)

### 4. Stale and Orphan Files

Check `results/metrics/` and `results/summaries/` for:
- Files that are NOT produced by any script in the current pipeline (orphans from deleted scripts)
- Files whose names suggest they belong to strategies that no longer exist in `src/` (e.g., strategy1_*, strategy3_* files if those scripts were deleted)
- The `.gitignore` or git status for large files that should not be committed

Check `data/derived/` for:
- Files referenced by scripts but not present (would cause runtime errors)
- Files present but not referenced by any script (stale derived data)

### 5. README Updates

Verify that `src/README.md` accurately describes:
- The current list of build scripts and their order
- The current list of analysis scripts
- The entry point (`run_all.py`) and its flags
- The data sources and their locations
- The shared modules (`_paths.py`, `_ols.py`)

Also check if there should be READMEs in:
- `data/raw/` (describing external data sources)
- `data/derived/` (describing derived datasets)
- `results/` (describing output structure)

### 6. Figure Generation

The three figures are generated by R scripts in `results/figures/`:
- `generate_fig1.R` → `fig1_fuel_vs_geo.pdf`
- `generate_fig2.R` → `fig2_calendar_time.pdf`
- `generate_fig3.R` → `fig3_world_map.pdf`

These depend on `results/summaries/event_level_betas.csv` (produced by `strategy2_robust_inference.py`) and data files in `data/derived/`. Verify:
- The R scripts are documented
- The CSV dependency is noted
- The R package requirements are listed (ggplot2, showtext, sysfonts, sf, rnaturalearth, rnaturalearthdata)
- The generated PDFs are copied to `manuscript/figures/` (currently manual; should this be automated?)

### 7. LaTeX Compilation

Verify that the manuscript compiles:
- `manuscript/main.tex` uses `\input{../results/tables/table_*.tex}` for appendix tables. Verify these files exist.
- `manuscript/references.bib` — verify all cited keys exist
- `manuscript/figures/` — verify all three PDFs exist and are referenced in the .tex
- Test compilation: `pdflatex main.tex && bibtex main && pdflatex main.tex && pdflatex main.tex`

## Output

Produce:
1. A dependency graph (text-based) showing the full pipeline from raw data to manuscript
2. A list of all gaps, stale files, and missing documentation
3. Specific commands or code changes to fix each issue

## Constraints
- Python 3.8+, stdlib-only + openpyxl
- R 4.5+ for figures (ggplot2, sf, rnaturalearth)
- MiKTeX for LaTeX compilation (Windows)
- Path resolution via `src/_paths.py` anchored to repository root
